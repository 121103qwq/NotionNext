# 维护路线图

本路线图用于追踪此下游仓库的维护工作；它不代表上游项目的发布计划。

## 近期目标

- [x] 建立问题、功能建议与拉取请求的统一提交模板。
- [x] 记录上游来源、当前可核实的差异及同步原则。
- [ ] 在配置远程仓库后，记录本仓库与上游的共同基线提交（SHA）与最近一次检查日期。
- [ ] 建立可重复的上游同步流程，并在每次同步后记录冲突处理和本地下游改动。
- [ ] 根据实际可运行的项目代码与依赖，补充本地验证命令和持续集成检查。

## 维护原则

1. 下游改动应保持可追踪：说明目的、关联问题和验证方式。
2. 同步上游前先检查本地下游改动，避免静默覆盖定制内容。
3. 不记录未经证实的使用数据、贡献数据或测试结论。
