# 上游与下游关系

## 声明的上游

本仓库是 `tangly1024/NotionNext` 的下游分支。该 GitHub 地址目前会重定向至
[`notionnext-org/NotionNext`](https://github.com/notionnext-org/NotionNext)；维护时应以该
重定向后的仓库作为上游项目的当前公开位置，并在实际配置远程后记录所使用的提交 SHA。

上游 NotionNext 是一个基于 Next.js 和 Notion API 的开源站点系统。关于上游的功能、
版本和维护政策，请以其仓库与发布说明为准。

## 当前可核实的下游差异

本地仓库在建立本基线时没有提交历史、没有配置 Git 远程，也没有项目源代码文件。因此，
无法可靠地计算相对上游的提交数、文件级差异或功能差异。

当前工作树中可确认的下游内容仅为本次维护基线新增的文件：

- `ROADMAP.md`：近期维护目标；
- 本文件：上游关系与差异记录；
- `.github/ISSUE_TEMPLATE/`：Bug Report 与 Feature Request 模板；
- `.github/pull_request_template.md`：Pull Request 模板。

除上述维护文件外，本仓库尚未包含可供比较的下游代码或配置，故不声称存在其他差异。

## 后续同步记录要求

首次导入或连接上游后，请在此处补充：上游 URL、共同基线 SHA、同步日期、保留的下游
改动、冲突处理及验证命令。不要使用未核实的统计数据替代这些记录。
